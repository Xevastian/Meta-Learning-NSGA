# Multi-Objective Optimization: Three Algorithm Baselines

This directory contains three complete MOO algorithm implementations:
- **NSGA-II**: With meta-learning (warm-start, adaptive operators)
- **MOEA/D**: Pure baseline (no meta-learning)
- **SMS-EMOA**: Pure baseline (no meta-learning)

## 1. NSGA-II (Non-dominated Sorting Genetic Algorithm II)

**File:** `nsga2.py`

**Algorithm:**
- Generational evolution: creates 2×pop_size offspring per generation
- Non-dominated sorting: ranks population by dominance levels
- Crowding distance: maintains diversity within each rank
- Tournament selection: selects parents based on rank + crowding distance

**Meta-Learning Features:**
- ✅ Warm-start: Initialize population from meta-knowledge database
- ✅ Adaptive operators: Mutation rate adjusts based on population diversity 
- ✅ Diversity tracking: Monitor diversity evolution per generation
- ✅ Meta-knowledge persistence: Log Pareto fronts for future runs

**Complexity:** O(N² × generations)

**Best for:** General-purpose optimization with meta-learning enhancements

## 2. MOEA/D (Multi-Objective Evolutionary Algorithm based on Decomposition)

**File:** `moead.py`

**Algorithm:**
- Generational evolution with decomposition
- Decomposition: Converts MOO problem into N scalar subproblems
- Neighborhood: Each subproblem collaborates with neighbors
- Aggregation: Uses weighted-sum or Tchebycheff scalarization

**Meta-Learning Features:**
- ❌ None (pure baseline algorithm)

**Complexity:** O(N × K × generations) where K = neighborhood size

**Best for:** Many-objective problems, diversity-focused search

## 3. SMS-EMOA (S-Metric Selection NSGA-II)

**File:** `sms_emoa.py`

**Algorithm:**
- Steady-state evolution: creates 1 offspring per generation
- Hypervolume-based selection: Removes individual causing minimal HV loss
- SBX crossover & polynomial mutation
- S-metric diversity: Uses hypervolume contribution for selection

**Meta-Learning Features:**
- ❌ None (pure baseline algorithm)

**Complexity:** Higher per-generation hypervolume computation, but fewer evaluations

**Best for:** Expensive fitness evaluations, quality-focused optimization

## Comparison Table

| Feature | NSGA-II | MOEA/D | SMS-EMOA |
|---------|---------|--------|----------|
| **Scalability** | Moderate | High | Low |
| **Simplicity** | High | Medium | Medium |

| **Speed** | Fast | Medium | Slow |

| **Meta-Learning** | ✅ Full | ❌ None | ❌ None |
| **Evolution Type** | Generational | Generational | Steady-state |

## Usage

### Run Individual Algorithm

```bash
# NSGA-II (with meta-learning)
python nsga2.py data.csv 20 10 --seed 42 --warm-start --adaptive

# MOEA/D (baseline, no meta-learning)
python moead.py data.csv 20 10 --seed 42

# SMS-EMOA (baseline, no meta-learning)
python sms_emoa.py data.csv 20 10 --seed 42
```

### Python Integration

```python
# NSGA-II with meta-learning
from nsga2 import nsga2
result = nsga2(
    pop_size=20,
    generations=10,
    data_path='data.csv',
    use_warm_start=True,
    adaptive_operators=True
)

# MOEA/D baseline (no meta-learning)
from moead import moead
result = moead(pop_size=20, generations=10, data_path='data.csv')

# SMS-EMOA baseline (no meta-learning)
from sms_emoa import sms_emoa
result = sms_emoa(pop_size=20, generations=10, data_path='data.csv')
```

### Run Comparison

```bash
python multi_algorithm_comparison.py data.csv --compare
```

## Algorithm Selection Guide

### Use NSGA-II when:
- You want meta-learning benefits (warm-start, adaptive operators)
- Problem size is moderate (≤ 100 decision variables)
- You want a simple, easy-to-understand algorithm
- Comparison with literature is important

### Use MOEA/D when:
- You need a pure baseline algorithm
- Problem has many objectives (≥ 3)
- You want structured neighborhood collaboration
- Diversity is critical

### Use SMS-EMOA when:
- You need a pure baseline algorithm
- Fitness evaluations are expensive
- Quality of Pareto front is paramount
- Steady-state evolution is preferred
